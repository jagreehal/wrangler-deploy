<span class="section-label">What you get</span>
<h2 class="section-title">Keeps your existing setup. Adds what's missing.</h2>

<div class="feature-rows">
  <div class="feature-cell">
    <h3>Works with your wrangler.jsonc</h3>
    <p>Point it at your existing config files. No rewriting, no data migration. <code>wrangler dev</code> still works. Already have Workers running? Run <code>wd introspect</code> to <a href="/wrangler-deploy/getting-started/migrating/">generate a config from your live account</a>.</p>
  </div>
  <div class="feature-cell">
    <h3>Types without codegen</h3>
    <p>Import your Env type directly from the config file. D1Database, KVNamespace, Queue types resolve at compile time. No generated files to commit.</p>
  </div>
  <div class="feature-cell">
    <h3>Shared state across the team</h3>
    <p>Store state in a Cloudflare KV namespace. One person creates the stage, another deploys to it, CI tears it down. Everyone sees the same thing.</p>
  </div>
  <div class="feature-cell">
    <h3>Production stages are protected</h3>
    <p><code>wd destroy --stage production</code> won't work without <code>--force</code>. PR stages get a TTL and auto-expire. Queue consumers are removed before workers, in the right order.</p>
  </div>
  <div class="feature-cell">
    <h3>Drift detection</h3>
    <p>Someone deleted a KV namespace from the dashboard? <code>wd status</code> tells you what changed. Compares local state against live resources.</p>
  </div>
  <div class="feature-cell">
    <h3>Secrets that block deploys</h3>
    <p>Declare which secrets each worker needs. If any are missing, deploy stops. Sync them from <code>.dev.vars</code> or set them interactively. They never touch state files.</p>
  </div>
</div>

<div class="cta-section">
  <span class="section-label">Get started</span>
  <h2 class="section-title">Three commands to a staging environment.</h2>
  <div class="install-cmd">
    <span class="dollar">$</span> npm install -D wrangler-deploy
  </div>
  <p class="section-desc" style="margin-inline: auto; text-align: center;">
    <strong>New project?</strong> Run <code>wd init</code> to scan your wrangler configs.<br/>
    <strong>Existing Workers?</strong> Run <code>wd introspect</code> to pull config from your live Cloudflare account.<br/>
    Then <code>wd apply --stage staging</code> to create everything.
  </p>
</div>
